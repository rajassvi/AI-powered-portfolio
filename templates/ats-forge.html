The Problem

Tailoring resumes is repetitive and unclear

📝

Manual resume tailoring is time-consuming — job seekers spend hours rewriting resumes for each application

🤖

Existing AI tools lack control — most resume builders generate generic outputs with no user refinement

🎯

ATS alignment is unclear — users don't know if their resume will pass automated screening systems

😰

Low confidence in resume quality — unclear which keywords matter and how to structure content effectively

The Solution

AI-assisted workflow with user control

✓

Designed step-based guided workflow that breaks complex resume creation into manageable phases

✓

Balanced AI assistance with manual control — users can edit, regenerate, and refine all AI outputs

✓

Implemented ATS compatibility scoring showing keyword coverage, skills match, and structure alignment

✓

Created iterative editing system allowing users to refine AI suggestions with real-time feedback

Understanding Users

Two Distinct User Needs

Users fall into two groups — those who need guidance and clarity, and those who prioritize speed and control. The product was designed to support both.

Design Principles

Key Decisions

01

Progressive Disclosure

Show only what's needed at each step.

02

Structured Inputs

Break fields into logical sections to reduce cognitive load.

03

Control Over Automation

AI assists, users decide. No black boxes.

04

Clarity Over Features

Do core tasks well vs adding complexity.

Reflection

Key Learnings

01

AI Needs Control

Full automation creates distrust. Users need editing capabilities and transparency.

02

Structure Reduces Overwhelm

Step-based flows help users complete complex tasks.

03

Reduce Decision Fatigue

Good UX isn't about fewer clicks — it's less mental effort.

04

Design for Both Users

Support beginners who need guidance and experts who need speed.

Final Takeaway

AI in UX requires balancing automation with control. Users need to trust the system, not just use it.
